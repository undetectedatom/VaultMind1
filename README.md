# VaultMind1

VaultMind1 is a backend service providing a powerful Retrieval-Augmented Generation (RAG) system built with **FastAPI**. It allows users to manage documents securely and interactively query knowledge bases using various large language models (LLMs).

## Features

- **FastAPI Core**: High-performance asynchronous API for seamless integration.
- **Advanced RAG Capabilities**: 
  - Supports **Hypothetical Document Embeddings (HyDE)** for improved retrieval.
  - Features dynamic query **routing**.
- **Multi-Provider Support**: Seamlessly configurable integration with multiple AI providers (OpenAI, Ollama, Doubao, etc.) via `rag_config.json`.
- **Document Management**: Supports ingestion of multiple formats including Markdown, HTML, PDF, Text, Word documents, and Images.
- **Chat & Session Handling**: Persistent user chat history, maintaining session context for continuous conversational querying.
- **Secure Authentication**: Built-in user authorization and authentication endpoints.

## Project Structure

- `app/`
  - `api/`: FastAPI routers mapping the endpoints (`/auth`, `/users`, `/documents`, `/chat`).
  - `database/`: Database engine configurations and SQLModel definitions.
  - `schemas/`: Pydantic models handling request and response validation.
  - `services/`: Core logic including the RAG pipeline, vector database interactions, and chat handling.
- `external_models/`: Modules managing interactions with specific external LLM APIs.
- `rag_config.json`: Central configuration file for routing, embedding, model selection, and generation strategies.

## Configuration

The application uses `rag_config.json` to configure the AI providers and RAG settings. You can define embeddings, base models, context windows, API bases, and feature toggles (like HyDE or Confidence Routing) in this file. API keys are injected via environment variables.

## Getting Started

1. Set up your environment variables (such as API keys in a `.env` file).
2. Configure `rag_config.json` according to the providers you want to use.
3. Start the application using a server like Uvicorn:
   ```bash
   uvicorn app.main:app --reload
   ```
4. Access the API documentation automatically generated by FastAPI at `http://127.0.0.1:8000/docs`.
